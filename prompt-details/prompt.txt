AI PDF Summarizer Agent - Project Specification
Overview
Build a simple but production-ready AI-powered PDF summarization tool.
The system should:

* Accept a PDF upload
* Extract and process text
* Generate a structured summary
* Extract actionable items (tasks, deadlines, responsibilities)
* Return a clean JSON response
* Display results in a simple frontend UI
Core Workflow

1. User uploads a PDF
2. Backend validates the file
3. PDF text is extracted
4. Text is split into chunks (1000–2000 tokens)
5. Each chunk is summarized individually
6. Chunk summaries are merged into a final summary
7. A separate agent extracts action items
8. Response is formatted as JSON
9. Frontend displays results
Tech Stack
Backend

* Node.js (Express) OR Python (FastAPI)
* PDF parsing: PyPDF / pdfplumber
* LLM API (OpenAI or equivalent)
Frontend

* Simple UI (React or plain HTML)
* File upload
* Display summary + action items
Required Features
1. Validation Layer

* Reject empty files
* Limit file size (e.g. 10MB)
* Handle parsing errors gracefully
2. Chunking System

* Split extracted text into chunks (1000–2000 tokens)
* Ensure chunks do not break sentences badly
3. Summarization Agent
For each chunk, generate:

* Short summary (2–3 sentences)
* Key bullet points
Then merge all chunk summaries into:

* Final concise summary
* Clean bullet list
4. Action Extraction Agent
From the FULL document summary, extract:

* Tasks
* Deadlines (if mentioned)
* Responsible parties (if mentioned)
LLM Prompt Design
Summarization Prompt

* Be concise
* Avoid repetition
* Focus on key information
* Output structured text
Action Extraction Prompt

* Only extract clear actionable items
* Do NOT hallucinate missing info
* Return structured list
Output Format (STRICT)
Return JSON in this exact format:
{ "summary": "string", "bullet_points": ["string", "string"], "actions": [ { "task": "string", "deadline": "string or null", "responsible": "string or null" } ] }
Frontend Requirements

* File upload component
* Loading state
* Display:
   * Summary
   * Bullet points
   * Action items
* Copy to clipboard button
Optional Enhancements (if time permits)

* Tone selection:
   * "Simple"
   * "Academic"
   * "Executive"
* Download result as JSON
Expectations

* Clean, modular code
* Clear separation of concerns
* Easy to extend
* Minimal but functional UI
Goal
Deliver a working MVP that can:

* Process real PDFs
* Produce useful summaries
* Extract meaningful actions
* Be ready for real-world usage
